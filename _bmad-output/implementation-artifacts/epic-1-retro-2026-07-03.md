# Retrospectiva - Epic 1: Base del Tenant y Servicios Núcleo

Fecha: 2026-07-03
Estado: Completa

## Epic Review

### Resultado

- Historias completadas: 11/11
- Alcance consolidado: identidad del tenant, branding, usuarios y permisos, WhatsApp, funnel, IA base, seguridad operativa, pagos, calendario, notificaciones y superadmin/auditoría

### Lo que funcionó

- Se mantuvieron fronteras claras por dominio.
- No se creó un `settings` genérico para resolver problemas que ya pertenecían a `companies`, `ai`, `integrations`, `payments`, `whatsapp` y `appointments`.
- El backend quedó como fuente de verdad para decisiones críticas y la UI se mantuvo como superficie de configuración y consulta.
- Las historias de integración siguieron un patrón de contrato explícito, validación tenant-scoped y pruebas de regresión.

### Fricciones y aprendizajes

- Los últimos cierres de review se concentraron en seguridad lateral: redacción de secretos, validación de proveedor correcto y rechazo de accesos cross-tenant.
- Las integraciones auxiliares tuvieron que dejar muy explícito que son best-effort y no pueden romper estados críticos ya confirmados.
- Cuando el contrato de auditoría no se redaccionó con suficiente rigor, aparecieron riesgos de persistir secretos en metadatos operativos.
- La historia de pagos mostró que un contrato compartido vale más que ramas separadas por proveedor.

### Lecciones clave

- Si una decisión afecta pagos, permisos, secretos o multi-tenancy, el contrato debe vivir en backend desde el primer día.
- La UI puede reflejar una superficie simple, pero no debe suavizar reglas que el backend necesita validar.
- Integraciones periféricas deben fallar de forma visible y aislada, nunca apropiarse de la verdad transaccional.
- Las historias fundacionales ganan mucho si cierran con pruebas de regresión sobre los puntos de borde, no solo sobre el flujo feliz.

### Acción tomada para cerrar la épica

- Se reforzaron guardrails de auditoría, redacción y aislamiento.
- Se consolidó el contrato de integración para pagos y calendario.
- Se dejó la base operativa lista para el trabajo conversacional del Inbox.
